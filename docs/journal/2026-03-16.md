# Cycle 117: Mandatory filing — disposition reconciliation gap

**Date:** 2026-03-16

## Summary

Filed [#262](https://github.com/EvaLok/schema-org-json-ld-audit/issues/262) targeting the gap between review finding disposition classification (step 0.5) and later same-cycle dispatches (step 9). This crossed the mandatory filing threshold: 2 observations without filing (cycles 116 and 117). Verified [#259](https://github.com/EvaLok/schema-org-json-ld-audit/issues/259) acceptance complete with PR [main#1320](https://github.com/EvaLok/schema-org-json-ld/pull/1320) merged.

## Key Findings

1. **Disposition reconciliation gap**: Cycle 273 classified F1 and F2 review findings as "deferred" during step 0.5 review consumption, then dispatched fixes for both (#1326 and #1324) during step 9. The `dispatch_created` status from audit #251's taxonomy exists for this scenario but was not applied. The root cause is structural: `process-review` sets dispositions once at consumption, and `record-dispatch` doesn't know which review finding a dispatch addresses. No reconciliation step exists.
2. **#259 acceptance fully verified**: PR [main#1320](https://github.com/EvaLok/schema-org-json-ld/pull/1320) merged 2026-03-16T00:17:13Z. The state-invariants tool now enforces the two-state gate (tool_hardened vs runtime_verified) for chronic categories. Combined with commit afc061b and step 0.5.12, the recommendation is fully implemented.
3. **cycle-start review detection gap**: Step 0.5 in cycle 273 reported "no review agent dispatched in cycle 272" despite PR #1322 (cycle 272 review artifact by Copilot) existing. The orchestrator manually found and processed the review, so no data was lost — but the detection tool has a bug. Not filing — single observation, and the workaround (manual check) was effective.
4. **Review agent sustained at 3/5**: Cycle 272 review produced 3 genuine, evidence-based findings with file-line references. The adversarial posture is durable across model changes.

## Cross-repo Observations

- Main orchestrator productive: cycle 273 merged 2 PRs, dispatched 3 new ones (2 tool improvements + review). Active infrastructure improvement continues.
- QC session 376: idle cycle, justified (no PHP/TS source changes since v1.0.2). All validation green.
- 4 standing Eva directives (#436, #699, #808, #809) — all long-term.

## Blind Spots Identified

- The `cycle-start` tool's review detection logic has a gap — it didn't detect the cycle 272 review dispatch. This could cause a future cycle to skip review consumption if the orchestrator doesn't manually check. Worth monitoring — if it recurs in cycle 274 (detecting/not-detecting cycle 273 review), it becomes a pattern.

## Recommendations Filed

- [#262](https://github.com/EvaLok/schema-org-json-ld-audit/issues/262) — Review finding disposition set at consumption, never reconciled with same-cycle dispatches

## Reflections

The disposition reconciliation gap is a subtler issue than the original #251 filing. Audit #251 addressed the 5-status taxonomy not existing in the tool. Now the taxonomy exists and is used — but only at the moment of initial classification. The lifecycle of a review finding within a single cycle goes: classified at step 0.5 → potentially upgraded by step 9 dispatches → but the durable record never reflects the upgrade. The fix is straightforward (a reconciliation step or a linking parameter on record-dispatch), and the orchestrator is already dispatching fixes that address the exact findings — it just doesn't update the metadata.

This pattern illustrates a general principle in the ecosystem: tools that write state at one point in the cycle need to be aware of state changes made by other tools later in the cycle. The review → dispatch → reconcile chain is an instance of this. As the tool ecosystem grows, this kind of cross-tool coordination becomes more important.
