## Blind Spots Identified

- **Observation-window discipline produces the right behavior in high-execution cycles.** This was the first cycle in the Opus 4.7 period where the audit could plausibly file 3-4 follow-on findings (audit #420 deferral rationale staleness, #2416/#2519 close-out lag, PR #2718 structural blind spot, polling-tool close-loop gap). - **The polling tool's signature-based detection design is interesting and worth noting.** [PR #2706](https://github.com/EvaLok/schema-org-json-ld/pull/2706) doesn't try to determine "is this comment from Eva?" by user login (the orchestrator runs as the EvaLok user via personal access token, so login-based detection would conflate orchestrator and human comments). Instead it uses signature-based detection: any comment NOT containing `> **[main-orchestrator]** | Cycle N` is treated as from Eva (the human). This is robust against orchestrator commenting under EvaLok but requires the orchestrator to maintain its own signature convention (which it does, per writing-conventions skill). Live invocation in cycle 540 confirmed zero false positives. Worth noting because it's the kind of design choice that becomes invisible after it works — but if it ever breaks (e.g., orchestrator forgets to sign a comment), the polling tool would surface a fake Eva response and cause unnecessary work.

- **The audit's role in this 24h window was clarifying, not generative.** The cycle 200 finding was the generative event; cycle 201's role is verification, accountability, and watch-item capture. Watching the orchestrator land 4 PRs traceable to a single audit observation, and having the discipline to record that and stop, is itself a behavior worth practicing. The Opus 4.7 + max-reasoning-effort orchestrator is producing high-quality structural action — the audit's value-add right now is selectivity, not volume.

- **The cycle 540 chronic-refresh-invalidation rollback is a process-quality milestone.** Cycle 539 made the same-cycle-bump-against-unmerged-PR error that the audit chain has been chasing for months. Cycle 540 review caught it. Cycle 540 enacted the rollback. Cycle 541 dispatched the journal-quality structural fix #2722. The end-to-end lag (defect → detection → rollback → structural dispatch) is now ~24h. Six months ago this would have been a 5-7 cycle process. The combination of (a) audit #411 ADR 0013 chronic-refresh-invalidation Tier 1 backstop, (b) max reasoning effort, (c) the polling-tool-driven Eva-response unblocking, has measurably tightened the self-correction loop.
