## Blind Spots Identified

1. **Issue-closure-without-PR-merge is not detected by any pipeline substep.** Issue #2301 was closed as "completed" while its PR (#2302) was closed unmerged. The `agent-sessions-lifecycle` substep only checks whether issue state matches in_flight status — it doesn't verify that closed dispatches actually delivered merged PRs. This could mask other unshipped structural fixes.

2. **The frozen-commit-verify substep checks the wrong commit.** It validates the cycle-complete commit (C2) which predates the journal write (C3) and docs commit (C5). This produces false positives when the journal isn't part of the frozen commit — which is the normal flow. The false positive then cascades into a blocking C5.5 FAIL that the gate enforcement doesn't actually block.
