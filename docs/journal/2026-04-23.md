## Reflections

Cycle 198 breaks the cycles-193-through-197 streak of zero filings under [#425](https://github.com/EvaLok/schema-org-json-ld-audit/issues/425) observation-window. Reading this as a signal: the observation-window directive is calibrated correctly. Over six cycles, the threshold held against 7 + 1 + 4 + 0 + 0 + 3 candidate observations (all properly journaled and allowed to self-resolve or self-disclose and dispatch); on cycle 198, a single structural novelty — the misdiagnosed self-disclosure — escaped the threshold and warranted filing. This is approximately the acceptance rate one would expect for a well-calibrated filter: most observations are correctly routed to journal, rare structural novelty earns a filing. The decline trajectory (cycle 194 declined 4, cycle 197 declined 7) suggests the filter has been doing real work even when no filing resulted.

The cycle 528 misdiagnosis is a specific kind of finding — the orchestrator's self-disclosure is correct about the event (it says "Cycle 528 close-out crashed silently") but wrong about the fix's coverage ("Cycle 529 dispatch #2657 mechanically prevents the recurrence"). This pattern — accurate symptom description, correct dispatch selection in spirit, but incorrect scope-of-coverage claim — may be underrepresented in the audit's historical filings. I've previously focused on cases where the orchestrator missed something entirely. Here the orchestrator saw the thing, filed a fix, and mis-bound the fix to the symptom. Worth watching whether this pattern recurs with other dispatches, because it's detectable by a specific check: "does the fix's commit receipts / acceptance criteria contain language that would mechanically address the observed failure mode?" The `commit_and_push` helper in PR #2658 is explicitly about commits-becoming-unpushed, not about session-termination-between-steps. A brief audit-side check of acceptance criteria language against failure symptoms could catch this class routinely.

The cycle-start race cluster closes out cleanly via PR #2658's eventual landing: Eva Q1+Q2 answered within 1h of cycle 197 close (fast response for her; the 196h on main#2519 makes the Q1+Q2 speed look like deliberate triage). The cycle 524 catastrophe spawned a structural response that addresses the broader class — all state-mutating tools must commit-and-push atomically, not just cycle-start. This is the right scope; a narrower fix would have left future bugs in the same class possible. Eva's framing ("Commit and push are atomic") makes the discipline a repo-wide invariant, which the `tools/git-reset-guard` wrapper enforces at read-time and `commit_and_push` enforces at write-time. A textbook root-cause fix. If PR #2658 lands cleanly in cycle 531, the cycle-start race cluster is retired.

The standing Eva-blocker count remains 10. Cumulative silence across those 10 is now 2,202 hours — 91.75 days of unanswered structural questions. Individually each has good reason (Eva triages; some are deeply architectural). Cumulatively, the main orchestrator operates with 10 concurrent gaps in its mental model of what the system should do. The cycle 197 journal already captured the concern; cycle 198 holds the line — no filing on cumulative pattern, because the observation-window directive is explicitly about not adding that kind of pressure. Noting: if the count reaches 12 or any single item crosses 400h, the proactive observation-window exception should be reconsidered.
