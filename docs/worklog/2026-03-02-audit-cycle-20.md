## What happened

### Input-from-eva #51 — cross-repo communication design

Eva's directive: the GitHub PAT bug that previously allowed cross-repo writes has been fixed. All orchestrators are now restricted to their own repos. Design a standard methodology (Claude skill) and tool that all three repos can adopt.

**Research phase**: Analyzed current communication patterns across all three repos:
- Main repo: Step 5 polls audit-outbound, comments directly on audit issues (NOW BROKEN), creates audit-inbound on own repo
- QC repo: Uses `tools/poll-repos.sh`, creates audit-inbound on own repo (already adapted to the constraint)
- Audit repo: Polls main/QC for activity but has NO step to discover audit-inbound responses (relied on direct comments)

**Key finding**: The main orchestrator discovered the PAT limitation at cycle 99: "Attempted to close feedback loop on audit repo #49 — PAT lacks write access to audit repo." But this was treated as a one-off, not a systematic process change.

**Design deliverable**: Filed [#53](https://github.com/EvaLok/schema-org-json-ld-audit/issues/53) — a `cross-repo-communication` Claude skill specification covering:
1. Core principles (write only to own repo, discover by polling, respond on own repo)
2. Standardized message types and labels
3. Communication flow diagrams
4. Polling templates per repo role
5. Standardized response issue format
6. State tracking schema (cross_repo_tracking section)
7. Lifecycle management rules
8. Tool recommendations (skill-only for sandboxed repos, script wrapper for others)

Both orchestrators will see this on their next polling cycles. Specific feedback questions included for each.
